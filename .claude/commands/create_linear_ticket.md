Workflow to create a linear ticket: 
Make sure Linear MCP is connected
1. base on $task_description 
2. Check if there is a similar ticket in linear and if so, return to confirm if still want to create a new ticekt. you can use find issue tool in linear MCP 
3. Make sure there is a owner in description 
4. find all the team, project, label, status priority, user names and cooresponding id
5. Try your best to create a ticket based on the content of $task_description with title, description, team, project, label, status priority, user names 
6. If you are not sure about the ticket, you can ask the user for confirmation. 
7. Create the ticket in linear and return the ticket url and git hub branch
